# 儿童数独乐园 — 设计文档

日期：2026-06-11
状态：用户授权自主补充需求，本文档记录全部需求决策。

## 目标

为 6 岁左右的孩子做一个网页版数独游戏：界面友好、操作简单、即时反馈、有成就感。
纯静态页面，双击 `index.html` 即可玩，无需安装、无需联网、无广告。

## 用户给定的需求

1. 棋盘大小可自由选择：4×4 到 9×9。
2. 符号可选：数字 或 图形（emoji，孩子容易理解）。

## 自主补充的需求决策

### 规则适配（关键决策）

| 棋盘 | 宫格划分 | 规则 |
|------|----------|------|
| 4×4 | 2×2 宫 | 行、列、宫不重复 |
| 5×5 | 无宫（质数无法均分） | 行、列不重复（拉丁方阵，儿童数独常见玩法） |
| 6×6 | 2 行 × 3 列宫 | 行、列、宫不重复 |
| 7×7 | 无宫 | 行、列不重复 |
| 8×8 | 2 行 × 4 列宫 | 行、列、宫不重复 |
| 9×9 | 3×3 宫 | 行、列、宫不重复 |

5×5 / 7×7 采用拉丁方阵规则而非异形宫（jigsaw），因为异形宫的视觉边界对 6 岁孩子太复杂。

### 符号主题（6 套）

- 数字 1-9
- 动物 🐶🐱🐰🦊🐻🐼🐸🐵🦁
- 水果 🍎🍌🍇🍓🍊🍉🍑🥝🍒
- 交通 🚗🚌🚒🚜✈️🚀🚁⛵🚲
- 海洋 🐳🐬🐠🐙🦀🐢🦈🐡🐚
- 小马 🦄：原创 SVG 卡通小马头像（应用户“小马宝莉”需求；官方角色图片有版权
  风险且需外部资源，故自绘原创形象，以身体色+鬃毛色组合区分，独角兽带角）

每个主题取前 N 个符号（N = 棋盘大小）。橡皮擦用自绘 SVG 图标
（Unicode 无橡皮 emoji，海绵 🧽 含义不直观）。

### 难度（3 档，按挖空比例）

- ⭐ 简单：挖空约 30%（6 岁起步）
- ⭐⭐ 普通：挖空约 45%
- ⭐⭐⭐ 挑战：挖空约 55%

所有谜题保证唯一解（生成后用解算器验证）。

### 交互（为幼儿优化）

- 点选格子 → 点选下方符号面板填入；再次点同符号可替换；橡皮擦清除。
- 符号面板是纯操作按钮，没有选中态（曾实现过“印章模式”，因模式状态对幼儿
  易造成混乱且引发过双重选中 bug，已移除）。未选格子时点符号，按钮轻摇并
  弹出“先点一个空格哦”气泡提示。
- 大按钮、大格子、圆角、高对比、明亮配色；支持触屏和鼠标。
- 即时温和纠错：填错（与正确答案不符）的格子轻轻摇晃并标红提示，不惩罚。
  （采用「对照答案」而非「找冲突」判错：谜题唯一解，判定更直接，对孩子反馈更明确。）
- 宫格边界呈现（迭代过四版）：细边框（太弱）→ 交替底色（仅靠颜色易混淆）→
  粗边框大间距容器（用户反馈行列连不上）→ 最终方案：DOM 按宫嵌套渲染，
  宫之间只隔一条 4px 细线（板底色透出），与格内 3px 间隙近乎等宽，
  行列视觉连贯，宫界靠线 + 底色区分。
- 同符号高亮 + 所选格子所在行/列高亮，帮助孩子观察。
- 💡 提示按钮：自动正确填入一格（每局不限次，鼓励而非考试）。
- ↩️ 撤销按钮。
- 完成时全屏彩纸 + 星星 + 欢呼音效 + 鼓励语。

### 声音

- Web Audio API 合成的简单音效（点击、填对、出错、胜利），无外部资源。
- 🔊/🔇 静音开关，状态存 localStorage。

### 成就感

- 每完成一局按难度得 1-3 颗星，累计星数存 localStorage，主页展示。

### 界面语言

- 简体中文，配大量 emoji 图标降低识字门槛。

## 技术方案

候选方案对比：

1. **纯静态 HTML/CSS/JS，普通 `<script>` 标签（选定）** — 双击即玩、file:// 下可用（ES Module 在 file:// 受 CORS 限制，故不用 module）、零依赖零构建。
2. Vite + 框架（React/Vue）— 对此规模过重，且需构建步骤。
3. 单文件 HTML 内联全部代码 — 可行但违背小文件高内聚原则，难维护。

### 文件结构

```
sudoku/
├── index.html
├── css/style.css
├── js/
│   ├── themes.js      # 符号主题配置
│   ├── boxes.js       # 各尺寸宫格划分
│   ├── solver.js      # 回溯解算器 + 解数量统计
│   ├── generator.js   # 完整盘生成 + 挖洞（唯一解校验）
│   ├── state.js       # 游戏状态（不可变更新）
│   ├── audio.js       # 音效
│   ├── confetti.js    # 庆祝动画
│   ├── ui.js          # 渲染棋盘/面板
│   └── main.js        # 装配与流程
├── tests/             # Node 内置 test runner 的单元测试
└── docs/superpowers/specs/  # 本文档
```

核心逻辑（boxes/solver/generator/state）采用 UMD-lite 写法（浏览器挂全局命名空间 `KidSudoku`，Node 下 `module.exports`），从而可用 `node --test` 做单元测试。

### 数据流

主题/尺寸/难度选择 → `generator` 产出 `{puzzle, solution}` → `state` 持有不可变状态（每次操作返回新状态，支持撤销）→ `ui` 渲染 → 用户操作派发到 `main` → 更新状态重绘 → 全部填对触发庆祝。

### 错误处理

- 生成器设重试上限，超时降级为减少挖空数，保证 1 秒内出题。
- localStorage 读写全部 try/catch（隐私模式下可用，只是不保存进度）。

### 测试

- 单元测试：boxes（宫格划分正确性）、solver（可解/不可解/唯一性计数）、generator（合法性、唯一解、挖空比例）、state（不可变性、撤销、胜利判定）。
- 手动验收：浏览器中跑通各尺寸 × 各主题，截图验证 UI。
